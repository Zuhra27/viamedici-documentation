    Mapping template

    General information

    The data in the datastore is indexed and made available based on concrete and non-dynamic mapping templates. (Further details regarding the dynamic: https://www.elastic.co/guide/en/elasticsearch/reference/current/dynamic.html)

    Caution:

    Because we use "dynamic", false additional keys (including their values) will be stored, but not indexed. This means that the key/value will be inside the _source, yet it won't be searchable.

    All defined and fixed keys are mapped to a concrete datatype on this basis. (Further details regarding field datatypes: https://www.elastic.co/guide/en/elasticsearch/reference/current/mapping.html#field-datatypes)

    The advantage of mapping is that the specific mapping produces more optimal results for search queries. This is mainly due to the fact that indexing does not take place automatically via Elasticsearch.

    Caution:

    Extensions of the mapping of the base indices must be done by Viamedici!

    In the following tables, the field type is also shown in addition to the basic data.

    Currently the following types are defined:

    • Must

    • Optional

    • Automatically (Created by a pipeline)

    • Customer-specific (Optional fields which are related to a customer specific logic)


    Info:

    Some of the customer-specific fields may be defined as optional in the future, provided that the data can be filled purely via standardized rules and mechanisms.

    Regular data indices

    Regular data is product information that is transferred to the InMemory Engine via the Versioning Engine function of EPIM. This is a snapshot of the current work data with the intention to publish it.

    CDX Fields

    To keep the data model of the Data Store as flexible as possible within the standard system, so-called Customer Defined Extension Fields as well as Customer Defined Extension Query Fields were introduced.

    Included!

    These fields are based on an array of objects. The array "cdxFields" and the array "cdxQueryFields" are included in all mappings of the following indices and are therefore available by default.

    The aim is to allow the possibility of customer-specific data to already be in the standard system. Since the cdx-data is part of the mapping, they have a defined data type and are searchable by default.

    This makes it very easy and convenient to enrich documents with specific data without having to customize the indexes.

    The cdxFields have to be used for data, which should be used or displayed by 3rd party tools, and the cdxQueryFields should only be used to add information for queries.
