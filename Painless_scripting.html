    Painless scripting

    Painless is the official scripting language of Elasticsearch, which is somehow similar to JavaScript.

    Related links and documentations:

    • Elasticsearch documentation: https://www.elastic.co/guide/en/elasticsearch/painless/current/painless-lang-spec.html

    With painless you can include some very powerful scripts within your query. The next query shows how to define some script_fields with the following functions:

    • substring: Get a number of characters based on the start position: substring(startPosition, numberOfChar).

    • split: Split the string and return the first and last part of the array as well as the complete array.

    Query with substring and split
    GET /customer_ds_hierarchies_deu_deu/_search
    {
    "query": {
    "match_all": {}
    },
    "script_fields": {
    "SubString": {
    "script": {
    "lang": "painless",
    "inline": "params._source.shortcut.substring(0,4)"
    }
    },
    "split": {
    "script": {
    "lang": "painless",
    "inline": "String[] parts = /-/.split(params._source.shortcut); return parts[0];"
    }
    },
    "splitArray": {
    "script": {
    "lang": "painless",
    "inline": "String[] parts = /_/.split(params._source.shortcut); return parts;"
    }
    },
    "lastSplitEntry": {
    "script": {
    "lang": "painless",
    "inline": "String[] parts = /_/.split(params._source.shortcut); return parts[parts.length-1];"
    }
    }
    },
    "stored_fields": [
    "_index",
    "_type",
    "_score",
    "_id",
    "_source"
    ],
    "_source": [
    "epimId",
    "uniqueId",
    "name",
    "langIso",
    "shortcut",
    "level",
    "seqordernr",
    "type",
    "hierarchies"
    ]
    }

    Key

    Description

    script_fields

    Requires the 'root' key with the concrete scripting conditions and objects.

    SubString,

    split,

    splitArray,

    lastSplitEntry

    Name of the key which will be created with the result.

    lang

    We define this to use the scripting language painless.

    source

    Desired scripting, logic and/or condition.

    stored_fields

    You have to define which stored fields should be included in the response next to the script_fields.

    _source

    Shrink the keys to the desired set within the result.

    Painless scripting, Query DSL details

    Note:

    If no stored_fields are defined, the _source, etc. are missing within the result!
