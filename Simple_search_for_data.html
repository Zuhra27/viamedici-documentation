    Simple search for data

    Searching for data, documents and information using the Search API is extremely easy and fast. However, depending on the requirements, the complexity of the queries can increase. Therefore, in this tutorial we slowly but surely want to get closer to the topic.

    First, we will start with a query for simply all data of an index. A query always starts with a GET followed by the name of the desired index. At the end we have to add /_search to 'activate' the search API, otherwise no search will be started.:

    GET [name of index]/_search


    It's allowed to use wildcards within the name, e.g.:

    GET [pattern of index]*/_search


    So far so good, let's do the first search. S imply get all documents of the index customer_ds_hierarchies_deu_deu:

    URI query
    GET /customer_ds_hierarchies_deu_deu/_search

    In addition to the actual data of the documents, we also receive general information from the search API:

    Key

    Description

    took

    Time in milliseconds for Elasticsearch to execute the search.

    timed_out

    Tells us if the search timed out or not.

    _shards

    Tells us how many shards were searched, as well as a count of the successful/failed searched shards.

    hits

    Search results.

    hits.total

    Total number of documents matching our search criteria.

    hits.hits

    Actual array of search results (defaults to first 10 documents).

    hits.hits._index

    Name of the index.

    hits.hits._type

    Name of the document type.

    hits.hits._id

    Id of the document.

    hits.hits._score

    Document score.

    hits.hits._scoure

    Array with the data of the document.

    Simple search for data, General information from the search API

    As an alternative to a search using URI, we can also search using a simple JSON structure or a JSON object. By the way: This kind of 'query language' is named Query DSL (Domain Specific Language).

    Query with JSON body
    GET /customer_ds_hierarchies_deu_deu/_search
    {
    "query": {
    "match_all": {}
    }
    }

    Key

    Description

    query

    Requires the 'root' key with the concrete query objects.

    match_all

    The simplest query, which matches all documents, giving them all a _score of 1.0.

    Simple search for data, Query DSL details

    So far so good, but a search for concrete content is more likely to be a typical scenario.

    Let's do a query for the word 'outdoor', once again begin with the URI variant:

    URI query
    GET /customer_ds_hierarchies_deu_deu/_search?q=outdoor

    Based on the Query DSL and the JSON object, we can define it slightly different and increase the size, to include more documents in our result:

    Query with JSON body
    GET /customer_ds_hierarchies_deu_deu/_search
    {
    "query": {
    "match": {
    "name": "outdoor tool"
    }
    },
    "size": 35
    }

    Key

    Description

    query

    Requires the 'root' key with the concrete query objects.

    match

    Query in one field.

    The next key/value defines the name of the key ("name") and the search string ("outdoor").

    size

    Number of documents within the JSON result (default is 10).

    Simple search for data, Query DSL details

    The query DSL offers us much more possibilities, so we will say goodbye to the URI variant at this point. At the same time, we extend our query by an object, which allows us to specify the desired operator. This is helpful when searching for several words:

    Query with JSON body
    GET /customer_ds_hierarchies_deu_deu/_search
    {
    "query": {
    "match": {
    "name": {
    "query": "outdoor indoor",
    "operator": "and"
    }
    }
    },
    "size": 35
    }

    Key

    Description

    query

    Requires the 'root' key with the concrete query objects.

    match

    Query in one field.

    Use an object to define the search strings as well as the operator to get different results.

    operator

    Valid operator: and | or (default).

    size

    Number of documents within the JSON result (default is 10).

    Simple search for data, Query DSL details

    We can use this kind of query to get all documents which have been created by the test system (locationCode is the value of the property 'location' of the PDK):

    Query with JSON body
    GET /customer_ds_hierarchies_deu_deu/_search
    {
    "query": {
    "match": {
    "locationCode": "TEST"
    }
    }
    }
