    Documents and Indices

    General information

    Elasticsearch is a distributed document store. Instead of storing information as rows of columnar data, Elasticsearch stores complex data structures that have been serialized as JSON documents.

    The JavaScript Object Notation (JSON) file format is a text-based, open standard format used to serialize and transfer structured data between a server and a Web application. The JSON format is easy for humans to read and write. Although it is based on a subset of the JavaScript programming language, it functions completely independent. The JSON format is smaller, faster, and easier to analyze than XML. These characteristics make the JSON format the ideal data exchange language.

    When it comes to querying or aggregating the documents, you must do it based on:

    • Indices

    • Documents

    Indices

    Index names must meet the following criteria:

    • Lowercase only.

    • Cannot include \,/,*,?,",<,>,|, ` ` (whitespace character), ,,#

    • Indices prior to 7.0 could contain a colon (:), but that’s been deprecated and won’t be supported in 7.0+.

    • Cannot start with -, _, +

    • Cannot be . or ..

    • Cannot be longer than 255 bytes (Note that it is bytes, so multi-byte characters will count towards the 255 limit faster).

    Documents

    General

    All documents consist mainly of fields and values.

    Documents and Indices, Data types in JSON format

    Documents must meet the following criteria:

    • The Id of the document is limited to 512 bytes in size, larger values will be rejected.

    • Only valid field types are allowed (see above).

    Data Store specific

    All indices within the Viamedici Data Store use a standardized mapping template. Therefore, the data types are given and pre-defined. Please refer to the mapping templates for details.

    The following table describes all standard field-value-combinations, all other values are related to the customer specific data within EPIM and can be documented in general:

    Field

    Valid values

    timestamp

    Format: YYYY-MM-DD'T'HH:mm:ssZ

    type

    hierarchy | product | variant | attribute | element | image | graphic | document | audio | video | text | productTable | editorialTable

    locationCode

    PROD | TEST | DEV | QA

    geoCode

    Static value per EPIM InMemory Engine, definition of the value is done based on https://plus.codes first 4 characters, e.g. 8FXC for Karlsruhe.

    langIso

    Static list of value based on EPIM and ISO codes: [language]_[country]

    Language in lowercase, country in uppercase, e.g.: deu_DEU, eng_GBR, ita_ITA.
